# Example outputs

These files show **what AutoLyrics produces** for one short clip, in every export
format the app supports: **LRC** (line + enhanced word-level), **SRT**, **ASS karaoke**,
and **JSON**.

> **These are illustrative samples, not a real recognition run.**
> The lyrics are an **original, generic, public-domain-style verse written by us**
> specifically for this demo — **no real or commercial song lyrics are used**. The
> word timings are hand-authored to be plausible and **internally consistent across all
> four formats** (the same word starts/ends drive the LRC tags, the SRT cues, the ASS
> `\k` syllables, and the JSON `words[]`). They were generated by running this little
> self-written `Result` through the project's actual exporters
> (`backend/pipeline/export.py`), so the formatting matches the real app byte-for-byte.
> See [`_gen.py`](./_gen.py) to regenerate.

When you run AutoLyrics on a real song, the values come from the pipeline
(Demucs vocal separation → faster-whisper transcription → optional torchaudio MMS
forced-alignment against your reference lyrics). The **shapes and formats are exactly
what you see here.**

## The demo clip

A ~31-second instrumental + vocal demo, vocals starting at ~9.2 s. Six lines:

```
Morning light across the bay
Soft and slow we drift away
Hum a tune the river knows
Watch how every ripple grows
Carry me where breezes call
Open skies above us all
```

`modeUsed` is `align` (forced alignment), because for this demo we provided the full
lyrics as reference — the near-perfect path where AutoLyrics only measures *when* each
known word occurs.

## Files

| File | Format | What it is |
|---|---|---|
| [`demo.lrc`](./demo.lrc) | LRC | Both a **line-level** block (`[mm:ss.xx]` per line) and an **enhanced word-level** block (`<mm:ss.xx>` before every word — the "gold sweep" karaoke highlight). |
| [`demo.srt`](./demo.srt) | SubRip | Standard subtitles: numbered cues with `HH:MM:SS,mmm --> HH:MM:SS,mmm` ranges. One cue per line. |
| [`demo.ass`](./demo.ass) | ASS karaoke | Advanced SubStation Alpha with full `[Script Info]` / `[V4+ Styles]` / `[Events]` headers and per-syllable `{\kNN}` karaoke tags (NN = syllable duration in centiseconds). This drives the word-by-word sweep in players like Aegisub / mpv / VLC. |
| [`demo.json`](./demo.json) | JSON | The full `Result` object — the app's native shape (see below), the most faithful export for re-importing or post-processing. |

## Format notes

### LRC — line vs. word

- **Line-level** is the classic format every lyric player understands:
  `[00:09.20]Morning light across the bay`. AutoLyrics exports this with `level=line`.
- **Enhanced (word-level)** adds an inline timestamp before each word and a closing
  tag at the line's end:
  `[00:09.20]<00:09.20>Morning<00:09.80>light…<00:12.10>`. Enhanced-LRC-aware players
  highlight each word as it's sung. AutoLyrics exports this with `level=word`.
- Time tags are `mm:ss.xx` (hundredths of a second).
- The `[ti:]/[ar:]/…` header tags at the top of `demo.lrc` are standard LRC metadata
  tags, shown here for completeness.

### SRT

- Cues numbered from 1, blank line between cues, trailing newline at end of file.
- Timestamps are `HH:MM:SS,mmm` (millisecond precision, comma decimal separator).
- SRT is line-level only (no per-word timing in the format itself).

### ASS karaoke

- Each line is one `Dialogue:` event whose `Text` field is a sequence of
  `{\kNN}syllable` runs. `NN` is that syllable's duration in **centiseconds**.
- Short `{\k6}` runs with no text absorb the ~0.06 s gaps **between** words so the
  karaoke clock stays in sync with the line's start/end. The `\k` durations across a
  line sum to the line's total span.
- ASS times are `H:MM:SS.cc` (centisecond precision).

### JSON — the `Result` shape

The native object the whole app speaks. Top level:

```jsonc
{
  "language": "en",
  "modeUsed": "auto" | "biasing" | "align",
  "segments": [
    {
      "id": 0,
      "start": 9.2,            // seconds
      "end": 12.1,             // seconds
      "text": "Morning light across the bay",
      "words": [
        { "start": 9.2, "end": 9.74, "word": "Morning", "prob": 0.98 }
        // start/end in seconds; prob = 0..1 confidence
      ]
    }
  ],
  "meta": {
    "modelSize": "large-v3",
    "separated": true,         // whether Demucs vocal separation ran
    "durationSec": 31.4,
    "engine": "faster-whisper"
  }
}
```

`words[].prob` is the per-word confidence (0–1); in the in-app Editor, low values get
the amber "check me" treatment. In this demo the lowest-confidence words are
*ripple* (0.86), *Hum* (0.88), and *breezes* (0.89) — the kind of held or breathy
syllables a recognizer is least sure about.
